Each guide is built around a job rather than a command: finding pages, fetching their content, working with whole archives, querying the columnar index, building a local dataset, looking up ranks, scanning the news feed, exploring the host graph, running a recrawl engine, scheduling recrawls by change rate, building a local search index, extracting content signals, and serving an API.
They assume you have run the [quick start](/getting-started/quick-start/).
